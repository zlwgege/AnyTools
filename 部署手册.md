# ToolBox 开发者工具门户 — 生产环境部署手册

> 版本：V0.1  
> 适用角色：运维工程师  
> 部署方式：单服务器 / Nginx + Node.js + SQLite

---

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 前端 | React 18 + Vite + TypeScript + Tailwind CSS |
| 后端 | Express + TypeScript + better-sqlite3 |
| 数据库 | SQLite（文件型，无需单独安装） |
| 构建产物 | 前端静态文件（`dist/`）+ 后端服务（`server/`） |
| 默认端口 | 前端 80/443（Nginx），后端 3001（Node.js） |

---

## 二、环境要求

### 2.1 服务器
- **操作系统**：Linux（推荐 Ubuntu 22.04 LTS / CentOS 8+）
- **CPU/Memory**：2核4G 起（根据并发调整）
- **磁盘**：20GB+（含日志和数据库增长空间）

### 2.2 软件依赖
```bash
# Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Nginx
sudo apt-get install -y nginx

# PM2（Node.js 进程守护）
sudo npm install -g pm2

# 验证
node -v   # v20.x
npm -v    # 10.x
nginx -v  # 1.18+
pm2 -v    # 5.x+
```

## 三、部署架构

```
用户 → Nginx (80/443)
       ├── /api/*  → 反向代理 → Node.js 后端 :3001
       └── /*      → 静态文件 → /opt/toolbox/dist/
```

## 四、部署步骤

### 4.1 创建部署目录

```bash
sudo mkdir -p /opt/toolbox
sudo chown $USER:$USER /opt/toolbox
cd /opt/toolbox
```

### 4.2 获取代码

将项目源码上传至 `/opt/toolbox/`，确保目录结构如下：

```
/opt/toolbox/
├── dist/                 # 前端构建产物（待生成）
├── server/               # 后端源码
│   ├── src/
│   ├── package.json
│   └── ...
├── src/                  # 前端源码
├── package.json
├── vite.config.ts
└── ...
```

### 4.3 前端构建

```bash
cd /opt/toolbox

# 安装前端依赖
npm ci

# 构建生产包（输出到 dist/）
npm run build
```

构建成功后，`dist/` 目录下应包含 `index.html` 和 `assets/`。

### 4.4 后端依赖安装

```bash
cd /opt/toolbox/server

# 安装后端依赖
npm ci
```

### 4.5 数据库说明

- 数据库文件：`/opt/toolbox/server/data.db`
- 首次启动时会**自动初始化表结构和种子数据**（2个默认用户）
- **生产环境务必做好数据库文件备份**

## 五、Nginx 配置

创建站点配置文件：

```bash
sudo vim /etc/nginx/sites-available/toolbox
```

写入以下内容（根据实际域名修改）：

```nginx
server {
    listen 80;
    server_name toolbox.example.com;   # <-- 修改为你的域名

    # 前端静态文件
    location / {
        root /opt/toolbox/dist;
        index index.html;
        try_files $uri $uri/ /index.html;   # 支持 React Router history 模式
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 60s;
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
}
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/toolbox /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 六、后端服务管理（PM2）

### 6.1 启动后端

```bash
cd /opt/toolbox/server

# 使用 PM2 启动
pm2 start npm --name "toolbox-server" -- run start

# 查看状态
pm2 status
pm2 logs toolbox-server
```

### 6.2 PM2  ecosystem 配置文件（推荐）

创建 `/opt/toolbox/server/ecosystem.config.cjs`：

```javascript
module.exports = {
  apps: [{
    name: 'toolbox-server',
    cwd: '/opt/toolbox/server',
    script: 'npm',
    args: 'run start',
    instances: 1,
    exec_mode: 'fork',
    env: {
      NODE_ENV: 'production',
      PORT: 3001,
      CORS_ORIGIN: '*',                # 生产建议改为具体域名，如 https://toolbox.example.com
    },
    error_file: '/var/log/toolbox/err.log',
    out_file: '/var/log/toolbox/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    merge_logs: true,
    max_memory_restart: '500M',
    restart_delay: 3000,
  }]
}
```

创建日志目录：

```bash
sudo mkdir -p /var/log/toolbox
sudo chown $USER:$USER /var/log/toolbox
```

使用配置文件启动：

```bash
cd /opt/toolbox/server
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup systemd   # 按提示执行，设置开机自启
```

## 七、环境变量说明

### 后端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | `3001` | 后端服务监听端口 |
| `CORS_ORIGIN` | `*` | 允许的跨域来源，生产环境建议设为具体域名 |
| `NODE_ENV` | — | 设为 `production` 可提升性能 |

### 配置示例

```bash
# 开发模式（允许所有跨域）
PORT=3001 CORS_ORIGIN="*" npm run start

# 生产模式（限制域名）
PORT=3001 CORS_ORIGIN="https://toolbox.example.com" NODE_ENV=production npm run start
```

## 九、数据库备份与恢复

### 9.1 备份

```bash
# 创建备份目录
sudo mkdir -p /opt/backups/toolbox

# 备份数据库（SQLite 直接复制文件即可）
cp /opt/toolbox/server/data.db /opt/backups/toolbox/data.db.$(date +%Y%m%d_%H%M%S)

# 查看备份列表
ls -lh /opt/backups/toolbox/
```

### 9.2 恢复

```bash
# 停止后端
pm2 stop toolbox-server

# 恢复数据库
cp /opt/backups/toolbox/data.db.20240115_120000 /opt/toolbox/server/data.db

# 重启后端
pm2 start toolbox-server
```

### 9.3 定时备份（crontab）

```bash
sudo crontab -e
```

添加：

```cron
# 每天凌晨 3 点备份数据库
0 3 * * * cp /opt/toolbox/server/data.db /opt/backups/toolbox/data.db.$(date +\%Y\%m\%d_\%H\%M\%S) && find /opt/backups/toolbox/ -name "data.db.*" -mtime +30 -delete
```

## 十三、常见问题

### Q1：前端页面刷新后 404
**原因**：Nginx 未配置 `try_files` fallback  
**解决**：确认 Nginx 配置中有 `try_files $uri $uri/ /index.html;`

### Q2：前端报错 "无法连接到 API"
**原因**：前端 API 地址未改为相对路径，或 Nginx 反向代理未生效  
**解决**：确认 `vite.config.ts` 中 API_BASE 为 `/api`，且 Nginx `/api/` location 配置正确

### Q3：数据库文件丢失
**原因**：部署时覆盖了 `server/data.db`  
**解决**：升级前务必备份数据库；生产环境建议将数据库文件放在独立目录（通过环境变量或修改 `db.ts` 中的路径）

### Q4：CORS 跨域报错
**原因**：生产环境域名与 CORS 配置不匹配  
**解决**：设置环境变量 `CORS_ORIGIN=https://你的域名.com`

### Q5：PM2 启动失败
```bash
# 查看详细错误
pm2 logs toolbox-server --lines 100

# 常见原因：端口被占用
sudo lsof -i :3001
sudo kill -9 <PID>
pm2 restart toolbox-server
```
